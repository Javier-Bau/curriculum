Servidor Linux endurecido

VPS · Linux · Seguridad · Hardening

En este proyecto documento el despliegue y securización de un servidor Linux.

Objetivo

Crear un servidor funcional y seguro, con superficie de ataque mínima.

Implementación

1. Acceso remoto seguro

SSH configurado solo con claves:

# Crear usuario sin privilegios
adduser usuario

# Copiar clave pública
mkdir /home/usuario/.ssh
nano /home/usuario/.ssh/authorized_keys
chmod 700 /home/usuario/.ssh
chmod 600 /home/usuario/.ssh/authorized_keys

# Deshabilitar root
nano /etc/ssh/sshd_config
# PermitRootLogin no
systemctl restart sshd

2. Configuración de firewall

# Activar UFW y abrir solo puertos necesarios
ufw default deny incoming
ufw default allow outgoing
ufw allow 22/tcp
ufw allow 80/tcp
ufw allow 443/tcp
ufw enable
ufw status verbose

3. Hardening y seguridad

# Fail2ban básico
apt install fail2ban
systemctl enable fail2ban
systemctl start fail2ban

# Actualizaciones automáticas
apt install unattended-upgrades
dpkg-reconfigure --priority=low unattended-upgrades

Resultado

Servidor operativo, consumo bajo y con principios de mínimo privilegio aplicados.
